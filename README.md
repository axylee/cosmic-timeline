# 🌌 Cosmic Timeline · 宇宙歷史時間軸

互動式時間軸，從宇宙誕生（138億年前）到今日。  
**線上版本：** https://axylee.github.io/cosmic-timeline/

---

## 檔案結構

```
cosmic-timeline/
├── index.html                # 主時間軸 UI
├── data/
│   └── events.json           # 所有事件、軸線、時代定義
├── tools/
│   └── cosmic-tools.html     # 工具箱（新增事件/軸線 + 圖片管理）
├── check.py                  # 驗證 events.json 的 Python 腳本
└── README.md
```

## 常見需求 → 操作步驟

### ─── 事件 ───────────────────────────────────────

#### 📍 查看事件詳情
1. 直接點擊時間軸上的圓點或 pill
2. Popup 顯示：父軸 › 軸線 ⤴ crossRef · 年份、描述、Wikipedia 連結

#### 🔍 搜尋並跳轉到事件
1. 點擊頂部搜尋框（或按鍵盤直接輸入）
2. 輸入中文名稱、英文名稱或年份
3. 下拉選擇結果，或按 Enter 跳轉第一個
4. 自動縮放並將該事件置中

#### ✏ 修改現有事件（圖片/名稱/描述/連結）
1. 點擊事件 → Popup 開啟
2. 點「✏ 編輯」
3. 修改任意欄位（圖片URL、中英文名稱、描述、Wikipedia連結）
4. 點「✓ 儲存修改」→ 立即生效
5. 點「⬇ 匯出 JSON」→ 下載 events.json
6. 覆蓋 `data/events.json` → 上傳 GitHub
> 💡 可以先修改多個事件，最後一次匯出，一個 JSON 包含所有修改

#### ✏ 修改事件的軸線 / CrossRef
1. 點擊事件 → Popup → ✏ 編輯
2. 「所在軸線」下拉：選擇要移動到的軸線
3. 「CrossRef」下拉：選擇縱線要連到哪條軸線（可選「— 無 —」清除）
4. 儲存修改 → 匯出 JSON → 覆蓋上傳

#### ➕ 新增事件
1. 用瀏覽器開啟 `tools/cosmic-tools.html`
2. 載入 `data/events.json`
3. 確認在「新增事件」分頁
4. 填寫：
   - **軸線定位**：所在軸線、顯示門檻、CrossRef
   - **基本資料**：中文名稱（必填）、英文名稱、開始年份（必填）、結束年份（有填=pill）、重要程度
   - **描述**：中英文描述
   - **連結**：中英文 Wikipedia URL
   - **圖片**：URL（選填，之後可用圖片管理補）
5. 點「✓ 加入事件」（可繼續加更多）
6. 點「⬇ 下載 events.json」
7. 覆蓋 `data/events.json` → 執行 `python check.py` → 上傳 GitHub

### ─── 軸線 ───────────────────────────────────────

#### 🔧 修改軸線顯示門檻（zoomMin）
> 顯示門檻決定軸線在什麼縮放程度下出現：0 = 全局都看得到，0.58 = 要 zoom in 才出現

**方法A（推薦）：透過事件 Popup**
1. 點擊該軸線上任意一個事件
2. 點「✏ 編輯」
3. 「顯示門檻」滑桿拖動（正下方說明：0 = 全局，0.58 = zoom in才出現）
4. 儲存修改 → 立即生效 → 匯出 JSON → 覆蓋上傳

**方法B：透過工具新增時設定**
1. `cosmic-tools.html` → 新增軸線 → 顯示門檻滑桿設定好再加入

#### 🔧 修改軸線的父軸
1. 點擊該軸線上任意事件 → ✏ 編輯
2. 「父軸」下拉：選擇新的父軸
3. 儲存修改 → 匯出 JSON → 覆蓋上傳

#### 🔧 修改軸線起始年份 / 消亡年份
> 起始年份決定分支線從哪個時間點開始畫；有消亡年份的文明線會在那一年終止

1. 點擊該軸線上任意事件 → ✏ 編輯
2. 「軸線起始年份」：填入年份（負數 = BC），決定分支起點
3. 「消亡年份」：填入年份（空白 = 至今）
4. 儲存修改 → 匯出 JSON → 覆蓋上傳

#### ➕ 新增軸線
1. `cosmic-tools.html` → 載入 JSON → 切換到「新增軸線」分頁
2. 填寫：
   - **ID**：英文小寫 + 連字號，不能重複（例：`korea`、`east-europe`）
   - **中文標籤**：顯示名稱
   - **父軸**：這條軸線從哪條分支出來
   - **分類群組**：natural / human / science / arts / religion / civilization / nation
   - **顏色**：用色票或直接輸入 hex
   - **顯示門檻**：0 = 全局，0.58 = zoom in才出現
   - **起始年份**（必填）/ **消亡年份**（選填）
3. 點「✓ 加入軸線」→ 新軸線立即出現在事件的軸線選單中
4. 繼續在「新增事件」分頁加事件到這條新軸線
5. 下載 JSON → 覆蓋上傳

### ─── 圖片 ───────────────────────────────────────

#### 🖼 補上單一事件的圖片（手動）
1. 點擊事件 → Popup → ✏ 編輯
2. 「圖片 URL」貼上圖片網址
3. 儲存 → 匯出 JSON → 覆蓋上傳

#### 🖼 批量補圖（自動搜尋）
1. `cosmic-tools.html` → 載入 JSON → 切換到「圖片管理」分頁
2. 點「列出缺圖事件」查看缺圖清單
3. 點「▶ 開始更新圖片」→ 自動從 Wikipedia / Wikidata / NASA 搜尋
   - 只找到一張候選圖 → 自動套用
   - 找到多張 → 暫停讓你手動選擇最佳圖片
4. 下載 JSON → 覆蓋上傳

#### 🔎 檢查失效圖片連結
1. `cosmic-tools.html` → 圖片管理 → 點「▶ 開始檢查」
2. 自動測試所有圖片連結是否可載入
3. 失效的可點「清除」逐一移除，或「⚠ 清除全部失效連結」
4. 下載 JSON → 覆蓋上傳

### ─── 顯示控制 ───────────────────────────────────

#### 👁 隱藏 / 顯示特定軸線
- 右側 Legend 面板 → **單擊**軸線名稱：切換顯示/隱藏（隱藏的軸線會有刪除線）

#### 🎯 Solo 模式：只看特定軸線
- **雙擊** Legend 中的軸線 → 進入 Solo，其他軸線淡化
- Solo 狀態下**單擊**其他軸線 → 加入比較（恢復亮度）
- **再雙擊** Solo 軸線 → 退出 Solo，恢復全部
- 左側「淡化強度」滑桿：調整非 Solo 軸線的透明度（1% ~ 30%）

#### 🌐 中英文切換
- 右上角「中 / EN」按鈕 → 切換所有文字（事件標籤、時代名稱、Popup 描述）
- 設定記憶在瀏覽器（localStorage）

## events.json 欄位說明

### axes（軸線）

| 欄位 | 必填 | 說明 |
|------|------|------|
| `id` | ✓ | 唯一識別碼（英文小寫 + 連字號） |
| `label` | ✓ | 顯示名稱 |
| `color` | ✓ | 顏色 hex（例：`#ef4444`） |
| `order` | ✓ | 排列順序（數字越小越靠上） |
| `parent` | | 父軸 id（null = 頂層） |
| `startYear` | ✓ | 分支線起始年份 |
| `endYear` | | 消亡年份（null = 至今） |
| `zoomMin` | ✓ | 顯示門檻（0 = 全局，0.58 = zoom in才出現） |
| `group` | ✓ | natural / human / science / arts / religion / civilization / nation |

### events（事件）

| 欄位 | 必填 | 說明 |
|------|------|------|
| `id` | ✓ | 唯一識別碼 |
| `year` | ✓ | 年份（負數 = BC） |
| `zh` | ✓ | 中文名稱 |
| `en` | | 英文名稱 |
| `axis` | ✓ | 所在軸線 id |
| `level` | ✓ | 重要程度（1=大 2=中 3=小） |
| `endYear` | | 結束年份（有填 = pill 事件，沒填 = 點事件） |
| `crossRef` | | 縱線連接目標軸線 id |
| `image` | | 圖片 URL |
| `desc_zh` | | 中文描述 |
| `desc_en` | | 英文描述 |
| `wiki_zh` | | 中文 Wikipedia URL |
| `wiki_en` | | 英文 Wikipedia URL |
| `category` | | 分類：civilization / politics / war / science / religion / culture / exploration / ... |
